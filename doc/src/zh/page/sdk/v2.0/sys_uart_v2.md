# SYS_UART API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 SYS_UART（系统调试串口）驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨平台兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成系统调试串口的初始化与数据输出。


## 概述
### 版本特性

SDK 2.0 版本的 SYS_UART 驱动基于统一 HAL_SYS_UART 接口实现，兼容标准 HAL 开发习惯。驱动支持系统调试串口自动初始化、单字符/字符串发送、时钟分频自动计算，接口简洁统一、调用流程固定，可在不同硬件板卡间无缝移植，专门用于系统日志打印、调试信息输出、printf 底层支撑等关键调试场景。

### 适用范围

适用于 SDK 支持的所有硬件板卡上的**系统调试串口（SYS_UART）**，包括启动日志打印、异常信息输出、printf 重定向、调试指令交互等。



## 头文件与依赖
### 核心头文件

```c title="sys_uart.c"
#include "hal_sys_uart.h"
```

*   **hal_sys_uart.h**：对外暴露标准 HAL_SYS_UART 操作接口，包含系统串口初始化、字符发送、字符串发送等核心函数，无额外依赖类型文件，接口轻量化。

### 编译依赖

*   **CONFIG_DRIVER_SYS_UART**：该宏用于在 Kconfig 中开启 SYS_UART 驱动编译，不定义此宏时，驱动不会被编译进工程。

*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于系统串口波特率分频系数自动计算，必须正确配置。

*   **CONFIG_UART_BAUD_RATE**：系统串口波特率配置（如 115200），驱动根据该值自动生成时钟分频。

	

## 核心数据类型

SYS_UART 为系统专用调试串口，接口高度精简，**无额外自定义枚举与结构体**，全部使用标准 C 数据类型，保证最小系统启动与打印的可靠性。



## HAL_SYS_UART 标准接口（hal_sys_uart.h）
### 系统串口初始化

```c title="hal_sys_uart.h"
void hal_sys_uart_init(void);
```

*   **功能**：初始化系统专用调试串口（SYS_UART），根据配置的 CPU 主频与目标波特率，自动计算并写入时钟分频系数，完成硬件初始化。
*   **参数说明**：无参数。
*   **调用说明**：
	* 必须在系统启动早期调用，通常位于 board_init 或 main 函数开头。
	* 调用后系统串口即可进入就绪状态，可直接发送数据。
	* 只需调用一次，无需重复初始化。

*   **底层说明**：驱动根据 `CONFIG_CPU_FREQ_MHZ` 和 `CONFIG_UART_BAUD_RATE` 自动计算分频值并写入 `REG_UART_0_CLKDIV`。

### 单字符发送

```c title="hal_sys_uart.h"
void hal_sys_putchar(char c);
```

*   **功能**：向系统调试串口发送单个字符，是最底层的发送接口，可用于实现 printf、日志单字节输出。
*   **参数说明**：
	* `c`：待发送的字符/字节数据，支持 ASCII 码与二进制数据。
*   **调用说明**：
	* 阻塞式发送，直接写入数据寄存器，硬件自动完成并串转换与发送。
	* 适用于调试、状态输出、协议帧字节发送等场景。
	* 是 `hal_sys_putstr` 与 `printf` 的底层基础接口。
*   **底层说明**：直接将数据写入 `REG_UART_0_DATA` 寄存器，硬件自动发送。

### 字符串发送

```c title="hal_sys_uart.h"
void hal_sys_putstr(char *str);
```

*   **功能**：发送以 `\0` 结尾的标准 C 字符串，循环调用 `hal_sys_putchar` 逐个发送字符，直到结束符停止。
*   **参数说明**：
	* `str`：字符串首地址，必须以 `\0` 结尾。
*   **调用说明**：
	* 常用于日志打印、提示信息、启动信息输出等场景。
	* 无需手动计算长度，自动识别字符串结束位置。
	* 可直接作为 `printf` 输出的底层实现。
*   **底层说明**：通过 while 循环遍历字符串，逐字节调用 `hal_sys_putchar` 完成发送。



!!! info "硬件通用限制说明"

    *   SYS_UART 为**系统专用调试串口**，通常对应硬件 UART0，专供系统日志与调试使用。
    *   波特率由 `CONFIG_UART_BAUD_RATE` 与系统时钟共同决定，初始化前必须正确配置。
    *   接口为最简阻塞式轮询模式，无中断、DMA、FIFO 配置，保证最小系统可正常打印。

	


!!! info "通用使用规则"

    *   **使用流程**：hal_sys_uart_init () 初始化 → 调用 hal_sys_putchar /hal_sys_putstr 发送数据。
    *   **初始化时机**：必须在使用任何发送函数之前完成初始化。
    *   **波特率统一**：整个系统运行期间波特率固定，不支持动态修改。
    *   **最小依赖**：SYS_UART 驱动无额外依赖，可在系统启动最早期使用，用于调试启动流程。
