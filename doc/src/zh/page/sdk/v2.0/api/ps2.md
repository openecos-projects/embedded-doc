# PS2 API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 PS2 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成 PS2 键盘 / 鼠标设备的控制与数据接收。



## 概述
### 版本特性

SDK 2.0 版本的 PS2 驱动基于统一 HAL_PS2 接口实现，兼容标准 HAL 开发习惯。驱动支持 PS2 控制器使能、控制配置、键盘扫描码读取、按键数据解析等功能，接口精简、稳定、轻量化，专门用于 PS2 键盘等人机交互设备。

### 适用范围

适用于 SDK 支持的所有板卡上的 PS2 设备通信场景，包括：

* PS2 键盘按键扫描码读取
* 工业键盘、矩阵键盘输入
* 嵌入式系统人机交互按键检测
* 键盘指令解析与功能映射



## 头文件与依赖
### 核心头文件

```c title="ps2.c"
#include "hal_ps2.h"
```

*   **hal_ps2.h**：对外暴露标准 HAL_PS2 操作接口，包含控制器配置、数据读取等核心函数。


### 编译依赖

*   **CONFIG_DRIVER_PS2**：在 Kconfig 中开启 PS2 驱动编译，不定义此宏时，驱动不会被编译进工程。
*   **CONFIG_DRIVER_GPIO**：用于 PS2 时钟与数据引脚的功能复用配置。
*   **CONFIG_DRIVER_SYS_UART**：用于测试时打印按键数据。



## 核心数据类型

PS2 驱动接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 语言内置数据类型，可直接调用。



## HAL_PS2 标准接口（hal_ps2.h）

### 设置 PS2 控制器控制寄存器

```c title="hal_ps2.h"
void hal_ps2_set_ctrl(uint32_t val);
```

*   **功能**：配置 PS2 硬件控制器工作模式，包括模块使能、接收使能、时钟控制、中断使能等。
*   **参数说明**：
	* `val`：32 位控制字，用于开启/关闭 PS2 控制器、配置接收模式。
*   **调用说明**：
	* 使用 PS2 前必须先调用该接口使能控制器。
	* 典型使能值：`0b11`（开启控制器 + 开启数据接收）。

### 读取 PS2 接收数据

```c title="hal_ps2.h"
uint32_t hal_ps2_get_data(void);
```

*   **功能**：从 PS2 控制器的接收 FIFO 中读取按键扫描码或设备数据。
*   **返回值**：
	* 0：无有效数据。
	* 非 0：有效的 PS2 按键扫描码。
*   **调用说明**：
	* 建议在循环中轮询调用。
	* 常见按键：`0x76` 表示 ESC 按键。




!!! info "硬件通用限制说明"

   *   PS2 为硬件原生解码控制器，无需 CPU 解析时序。
   *   读取的数据为标准 PS2 键盘扫描码。
   *   必须正确配置 GPIO 复用功能，否则无法通信。
   *   无数据时返回 0，有数据时返回非 0 有效码。

	



!!! info "通用使用规则"

   *   **使用流程**：GPIO 复用配置 → hal_ps2_set_ctrl 使能 → 循环 hal_ps2_get_data 读取。
   *   引脚配置必须正确，否则无法接收数据。
   *   轮询读取时可适当增加延时，降低 CPU 占用。
   *   按键断开码通常以 0xF0 开头。
