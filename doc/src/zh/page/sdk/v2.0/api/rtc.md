# RTC API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 RTC 实时时钟驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成时钟控制、预分频配置、计数器读写、闹钟设置、中断状态获取等功能。



## 概述
### 版本特性

SDK 2.0 版本的 RTC 驱动基于统一 HAL_RTC 接口实现，兼容标准 HAL 开发习惯。RTC为硬件实时时钟单元，提供独立计时、秒脉冲产生、计数器递增、闹钟触发、中断标志等功能。驱动接口精简、功耗低、可后台持续运行，适用于时间计时、定时唤醒、超时检测、日历时钟等场景。

### 适用范围

适用于 SDK 支持的所有板卡上的 RTC 应用场景，包括：

* 系统实时时间计时
* 低功耗定时唤醒
* 闹钟/超时中断触发
* 秒脉冲、固定周期事件产生
* 系统时间戳记录




## 头文件与依赖
### 核心头文件

```c title="rtc.c"
#include "hal_rtc.h"
```

*   **hal_rtc.h**：对外暴露标准 HAL_RTC 操作接口，包含控制、预分频、计数器、闹钟、状态获取等核心函数。

### 编译依赖

*   **CONFIG_DRIVER_RTC**：该宏用于在 Kconfig 中开启 RTC 驱动编译，不定义则不编译。
*   **CONFIG_CPU_FREQ_MHZ**：用于 RTC 时钟分频计算，保证计时精度。



## 核心数据类型

RTC 驱动接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 32 位数据类型，轻量、稳定、上电即可使用。



## HAL_RTC 标准接口（hal_rtc.h）
### 设置 RTC 控制寄存器

```c title="hal_rtc.h"
void hal_rtc_set_ctrl(uint32_t val);
```

*   **功能**：配置 RTC 控制器，包括模块使能、计数器递增使能、闹钟中断使能、触发模式选择等。
*   **参数说明**：
	* `val`：32 位控制字。
*   **调用说明**：
	* 例：`hal_rtc_set_ctrl(1)` 使能 RTC 核心。
	* 例：`hal_rtc_set_ctrl(0b0010010)` 使能计数器递增 + 闹钟触发。

### 设置 RTC 预分频系数

```c title="hal_rtc.h"
void hal_rtc_set_prescale(uint32_t val);
```

*   **功能**：设置 RTC 时钟预分频值，用于控制计数器递增频率，决定计时精度。
*   **参数说明**：
	* `val`：预分频系数，数值越大计数越慢，计时周期越长。
*   **调用说明**：
	* 典型配置：`hal_rtc_set_prescale(48)`，用于生成 1ms 或 1s 基础时基。

### 设置 RTC 计数器初始值

```c title="hal_rtc.h"
void hal_rtc_set_cnt(uint32_t val);
```

*   **功能**：手动设置 RTC 计数器（CNT）的初始值，用于时间校准、从零开始计时。
*   **返回值**：
	* `val`：32 位计数器初始值。
*   **调用说明**：
	* 设置后立即生效，硬件从该值开始递增。

### 获取当前 RTC 计数器值

```c title="hal_rtc.h"
uint32_t hal_rtc_get_cnt(void);
```

*   **功能**：读取 RTC 实时计数器当前值，反映当前已运行时间。
*   **返回值**：32 位无符号整数，表示当前计数值。
*   **调用说明**：
	* 可用于计算运行时长、时间戳获取、超时判断。

### 设置 RTC 闹钟比较值

```c title="hal_rtc.h"
void hal_rtc_set_alrm(uint32_t val);
```

*   **功能**：设置闹钟比较值，当计数器 CNT 等于该值时触发闹钟标志/中断。
*   **参数说明**：
	* `val`：闹钟目标计数值。
*   **调用说明**：
	* 通常设置为 `当前cnt + 偏移值`，例：`hal_rtc_set_alrm(hal_rtc_get_cnt() + 10)`。

### 获取 RTC 中断状态标志

```c title="hal_rtc.h"
uint32_t hal_rtc_get_ista(void);
```

*   **功能**：读取 RTC 中断状态寄存器，判断闹钟是否触发、计数器是否更新完成。
*   **返回值**：
	* 0：无事件
	* 1：闹钟触发/计数器递增完成
*   **调用说明**：
	* 用于轮询等待闹钟事件。
	* 无需清除，硬件自动同步。




!!! info "硬件通用限制说明"

   *   RTC 为独立硬件计时器，可在低功耗模式下运行。
   *   计时精度由预分频系数决定。
   *   闹钟为单次比较匹配触发。
   *   计数器为 32 位向上递增模式。




!!! info "通用使用规则"

   *   **使用流程**：set_ctrl 使能 → set_prescale 分频 → set_cnt 初始值 → 循环 get_cnt /get_ista。
   *   闹钟需设置在当前计数值**之后**，否则无法触发。
   *   轮询递增完成时需等待 `ista == 1`。
   *   预分频值根据时钟频率调整，确保计时准确。
