# WDG API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 WDG（看门狗）驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成看门狗控制、预分频、比较值、喂狗、密钥解锁、状态获取等功能，保障系统稳定运行、防止死机卡死。



## 概述
### 版本特性

SDK 2.0 版本的 WDG 驱动基于统一 HAL_WDG 接口实现，兼容标准 HAL 开发习惯。WDG 为硬件看门狗模块，支持**密钥保护机制**、可配置预分频、比较值超时、喂狗刷新、复位触发等功能，具备高可靠性、强制复位、防跑飞特性，是嵌入式系统必备的稳定保障模块。

### 适用范围

适用于 SDK 支持的所有板卡上的看门狗应用场景，包括：

* 系统死机自动复位
* 程序跑飞保护
* 嵌入式设备稳定性保障
* 无操作系统环境下的异常处理
* 工业控制、车载设备等强可靠性场景



## 头文件与依赖
### 核心头文件

```c title="wdg.c"
#include "hal_wdg.h"
```

*   **hal_wdg.h**：标准 HAL_WDG 操作接口，包含控制、分频、比较值、喂狗、密钥、状态获取等全套函数。

### 编译依赖

*   **CONFIG_DRIVER_WDG**：该宏用于在 Kconfig 中开启 WDG 驱动编译，不定义则不编译。
*   **CONFIG_CPU_FREQ_MHZ**：用于看门狗超时时间计算。




## 核心数据类型

WDG 驱动接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 32 位数据类型，稳定、安全、可直接调用。




## HAL_WDG 标准接口（hal_wdg.h）
### 设置 WDG 控制寄存器

```c title="hal_wdg.h"
void hal_wdg_set_ctrl(uint32_t val);
```

*   **功能**：配置看门狗工作模式，包括模块使能、中断/复位选择、计数模式等。
*   **参数说明**：
	* `val`：32 位控制字。
*   **调用说明**：
	* 例：`0b101` 表示使能看门狗 + 开启复位触发。
	* **所有配置操作必须先解锁密钥**。

### 设置 WDG 预分频系数

```c title="hal_wdg.h"
void hal_wdg_set_prescale(uint32_t val);
```

*   **功能**：设置看门狗时钟分频，决定看门狗计数速度。
*   **参数说明**：
	* `val`：预分频值，通常写为 `目标分频 - 1`。
*   **调用说明**：
	* 分频越大，看门狗超时时间越长，例：`100-1` = 100 分频。

### 设置 WDG 比较值

```c title="hal_wdg.h"
void hal_wdg_set_cmp(uint32_t val);
```

*   **功能**：设置看门狗超时比较值，计数达到该值触发复位。
*   **参数说明**：
	* `val`：比较值，通常写 `目标值 - 1`。
*   **调用说明**：
	* 数值越大，超时时间越长，例：`500-1` 表示计数 500 个周期后触发。

### 喂狗（刷新看门狗）

```c title="hal_wdg.h"
void hal_wdg_feed(uint32_t val);
```

*   **功能**：喂狗操作，重置看门狗计数器，防止系统复位。
*   **参数说明**：
	* `val`：喂狗命令值，通常为 0/1 或特定序列。
*   **调用说明**：
	* 程序正常运行时必须周期性调用。
	* 超时未喂狗 → 系统自动复位。

### 设置 WDG 解锁密钥

```c title="hal_wdg.h"
void hal_wdg_set_key(uint32_t val);
```

*   **功能**：写入看门狗保护密钥，解锁寄存器配置权限。
*   **参数说明**：
	* `val`：密钥值，硬件固定为 `0x5F3759DF`。
*   **调用说明**：
	* **修改任何寄存器前必须先解锁**。
	* 防止误操作导致看门狗异常。

### 获取 WDG 状态

```c title="hal_wdg.h"
uint32_t hal_wdg_get_stat(void);
```

*   **功能**：获取看门狗当前状态，判断是否触发复位/超时。
*   **返回值**：
	* 0：未触发
	* 1：已触发复位/超时
*   **调用说明**：
	* 可用于轮询等待看门狗触发。




!!! info "硬件通用限制说明"

   *   所有寄存器**受密钥保护**，未解锁无法配置。
   *   密钥固定为 `0x5F3759DF`，不可修改。
   *   超时未喂狗 → 强制芯片复位。
   *   一旦使能，部分硬件不支持关闭。





!!! info "通用使用规则"

   *   **任何配置前必须先解锁**：`hal_wdg_set_key(0x5F3759DF)`。
   *   使用流程：解锁 → 配置 → 使能 → 循环喂狗。
   *   喂狗必须**周期性调用**，不能超时。
   *   超时时间 = 分频 × 比较值 / 时钟。
