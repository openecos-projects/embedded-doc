# I2C API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 I2C 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨平台兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成 I2C 主机通信、设备读写、时序控制等功能。



## 概述
### 版本特性

SDK 2.0 版本的 I2C 驱动基于统一 HAL_I2C 接口实现，兼容标准 HAL 开发习惯。驱动支持多端口选择、时钟分频配置、8 位 / 16 位寄存器地址、多字节连续读写、自动 ACK 检测、起始位 / 停止位控制，接口统一规范、参数清晰、可在不同硬件平台间无缝移植，专为传感器、EEPROM、RTC、IO 扩展等低速外设提供稳定可靠的通信能力。

### 适用范围

适用于 SDK 支持的所有平台上的 I2C 主机通信场景，包括传感器数据读取、EEPROM 读写、RTC 时钟配置、IO 扩展器控制、屏幕与外设寄存器配置等。



## 头文件与依赖
### 核心头文件

```c title="i2c.c"
#include "hal_i2c.h"
#include "hal_i2c_type.h"
```

*   **hal_i2c.h**：标准 HAL_I2C 操作接口，包含初始化、去初始化、多字节写、多字节读等核心函数。
*   **hal_i2c_type.h**：定义 I2C 相关枚举、配置结构体、端口类型，提供强类型安全，避免非法参数传入。



### 编译依赖

*   **CONFIG_DRIVER_I2C**：该宏用于在 Kconfig 中开启 I2C 驱动编译，不定义此宏时，驱动不会被编译进工程。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于 I2C 时钟分频与时序计算，必须正确配置以保证通信稳定。



## 核心数据类型（hal_i2c_type.h）
### I2C 端口枚举

```c title="hal_i2c_type.h"
typedef enum{
    HAL_I2C_PORT_0 = 0,
    HAL_I2C_PORT_MAX
} hal_i2c_port_t;
```

*   **功能**：用于标识硬件 I2C 控制器端口编号，支持多端口扩展。
*   **使用说明**：默认支持 `HAL_I2C_PORT_0`，具体可用端口由硬件决定。

### I2C 寄存器地址长度枚举

```c title="hal_i2c_type.h"
typedef enum{
    HAL_I2C_REG_8  = 8,
    HAL_I2C_REG_16 = 16
} hal_i2c_reg_addr_len_t;
```

*   **HAL_I2C_REG_8**：8 位寄存器地址，适用于大部分简单外设（传感器、IO 扩展）。
*   **HAL_I2C_REG_16**：16 位寄存器地址，适用于大容量 EEPROM、复杂外设。
*   **使用说明**：需根据目标设备的数据手册选择对应地址长度。

### I2C 配置结构体

```c title="hal_i2c_type.h"
typedef struct{
    uint32_t pscr;  // 时钟预分频系数
} hal_i2c_config_t;
```

*   **pscr**：预分频系数，对系统时钟分频，生成 I2C 通信时钟。
*   **使用说明**：数值由系统时钟与目标 I2C 速率共同决定，需匹配外设最高支持频率。



## HAL_I2C 标准接口（hal_i2c.h）
### I2C 初始化

```c title="hal_i2c.h"
int hal_i2c_init(hal_i2c_port_t port, const hal_i2c_config_t *config);
```

*   **功能**：初始化指定 I2C 端口，配置时钟分频、复位控制器、使能 I2C 硬件模块，自动完成引脚复用配置。
*   **参数说明**：
	* `port`：I2C 端口号，使用 `hal_i2c_port_t` 枚举，如 `HAL_I2C_PORT_0`。
	* `config`：指向 `hal_i2c_config_t` 结构体的指针，包含时钟分频配置。

*   **返回值**：0 成功，负数表示参数非法或初始化失败。
*   **调用说明**：
	* 系统上电后调用一次，完成硬件初始化与时钟配置。
	* 自动复位 I2C 控制器，清除异常状态，保证总线进入空闲。
	* 必须在所有读写接口之前调用。

### I2C 去初始化

```c title="hal_i2c.h"
int hal_i2c_deinit(hal_i2c_port_t port);
```

*   **功能**：关闭指定 I2C 端口，关闭时钟、禁用控制器、释放硬件资源，用于低功耗或资源重分配。
*   **参数说明**：
	* `port`：目标 I2C 端口编号。

*   **返回值**：0 成功，负数失败。

### 多字节连续写

```c title="hal_i2c.h"
int hal_i2c_write_nbyte(hal_i2c_port_t port, uint8_t slave_addr, uint16_t reg_addr, hal_i2c_reg_addr_len_t reg_addr_len, const uint8_t* data, uint32_t len);
```

*   **功能**：向目标 I2C 设备执行完整写操作：起始位 → 从机地址（写）→ ACK → 寄存器地址 → 连续写入数据 → 停止位。
*   **参数说明**：
	* `port`：I2C 端口编号。
	* `slave_addr`：7 位从机设备地址（不含读写位）。
	* `reg_addr`：目标寄存器起始地址。
	* `reg_addr_len`：寄存器地址长度（8 位 / 16 位）。
	* `data`：写入数据缓冲区指针。
	* `len`：写入数据长度（字节）。

*   **返回值**：0 成功，负数失败。
*   **调用说明**：
	* 内部自动处理起始位、停止位、ACK 检测。
	* 16 位地址自动先发高 8 位，再发低 8 位。
	* 适合寄存器配置、数据存储等场景。

### 多字节连续读

```c title="hal_i2c.h"
int hal_i2c_read_nbyte(hal_i2c_port_t port, uint8_t slave_addr, uint16_t reg_addr, hal_i2c_reg_addr_len_t reg_addr_len, uint8_t* data, uint32_t len);
```

*   **功能**：从目标 I2C 设备执行完整读操作：伪写寻址 → 重启信号 → 从机地址（读）→ 连续读取数据 → 最后字节 NACK → 停止位。
*   **参数说明**：
	* `port`：I2C 端口编号。
	* `slave_addr`：7 位从机设备地址。
	* `reg_addr`：目标寄存器起始地址。
	* `reg_addr_len`：寄存器地址长度（8 位 / 16 位）。
	* `data`：写入数据缓冲区指针。
	* `len`：写入数据长度（字节）。

*   **返回值**：0 成功，负数失败。
*   **调用说明**：
	* 自动处理 I2C 读协议的 “先写后读” 流程。
	* 自动控制 ACK / NACK 时序：倒数第 2 字节发 ACK，最后 1 字节发 NACK。
	* 适合传感器数据、状态寄存器、EEPROM 读取等场景。



!!! info "内部核心机制（驱动原理）"

    *   起始位与从机地址发送：驱动内部自动发送起始信号，并将 7 位地址左移 1 位，补上读写位，等待从机 ACK 应答，无 ACK 则自动重试。
    *   ACK 应答检测：每发送一个字节，都会等待硬件 ACK 标志，确保从机正确接收，避免通信丢包。
    *   时钟时序控制：通过 `pscr` 预分频系数配置 SCL 时钟，严格匹配 I2C 协议时序。
    *   总线忙等待：所有操作前都会检查 `BUSY` 标志，确保总线空闲后再操作，避免多主机冲突。




!!! info "硬件通用限制说明"

    *   驱动为主机模式，**不支持从机模式**。
    *   支持 **7 位从机地址**，不支持 10 位地址扩展。
    *   时钟速率由 `pscr` 分频决定，必须在外设支持的频率范围内。
    *   同一总线可挂载多个设备，通过不同 `slave_addr` 区分。
    *   通信异常时会自动等待总线释放，避免总线卡死。

	


!!! info "通用使用规则"

    *   **标准流程**：hal_i2c_init → 读写操作 → hal_i2c_deinit。
    *   **地址规则**：传入 7 位设备地址，驱动内部自动处理读写位。
    *   **时序规则**：尽量避免频繁短字节通信，防止总线拥塞。
    *   **异常处理**：通信失败时检查设备地址、时钟分频、硬件接线。
