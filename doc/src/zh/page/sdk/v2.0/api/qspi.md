# QSPI API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 QSPI 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨平台兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成高速同步串行通信控制。



## 概述
### 版本特性

SDK 2.0 版本的 QSPI 驱动基于统一 HAL_QSPI 接口实现，兼容标准 HAL 开发习惯。驱动支持时钟分频配置、多端口选择、多片选信号控制、8/16/32 位单次发送、批量数据发送等功能，所有接口统一规范、参数清晰、可在不同的板卡间无缝移植，专为高速外设通信、Flash 读写、显示屏控制等高速同步串行场景提供稳定底层支撑。

### 适用范围

适用于 SDK 支持的所有板卡上的 QSPI 高速同步通信场景，包括片外存储器读写、高速外设控制、显示模块通信、大容量数据批量传输等。



## 头文件与依赖
### 核心头文件

```c title="timer.c"
#include "hal_qspi.h"
```

*   **hal_qspi.h**：标准 HAL_QSPI 操作接口，包含初始化、去初始化、单字/批量数据发送、片选控制等核心函数，全部使用标准 C 基础数据类型。

### 编译依赖

*   **CONFIG_DRIVER_QSPI**：该宏用于在 Kconfig 中开启 QSPI 驱动编译，不定义此宏时，驱动不会被编译进工程。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于 QSPI 通信时钟的分频系数计算，必须正确配置才能保证通信时序的稳定。



## 核心数据类型

!!! info "核心数据说明"

    本模块不单独提供枚举、结构体头文件，端口、片选、配置参数直接在 `hal_qspi.h` 内定义，全部采用标准基础数据类型，精简高效：

    * **端口标识**：使用 `uint8_t` 类型，0 代表 QSPI 主控制器端口，有效端口范围由目标平台硬件决定。
    * **片选标识**：使用 `uint32_t` 类型，通过位域定义多片选信号，支持多外设挂载同一 QSPI 总线。
    * **时钟分频**：使用 `uint32_t` 类型，直接传入分频系数，通过`QSPI时钟 = 总线时钟 / (2 × (分频值 + 1))`来计算通信速率。





## HAL_QSPI 标准接口（hal_qspi.h）
### QSPI 初始化

```c title="hal_qspi.h"
int hal_qspi_init(uint8_t port, uint32_t clkdiv);
```

*   **功能**：初始化指定 QSPI 控制器端口，配置时钟分频系数，复位硬件控制器状态，清空发送、接收寄存器，使 QSPI 进入就绪通信状态。
*   **参数说明**：
	* `port`：QSPI 控制器端口编号，标准平台默认使用 0，有效端口由具体硬件板卡决定。
	* `clkdiv`：时钟分频系数，用于配置 QSPI 通信时钟频率，需匹配外接设备的最高支持时钟。

*   **返回值**：0 表示操作成功，非 0 表示参数非法、硬件初始化失败。
*   **调用说明**：
	* 系统上电后仅需调用 1 次，完成硬件基础初始化。
	* 初始化后自动清空硬件缓存，进入标准同步主机通信模式。
	* 必须在所有数据发送接口调用前执行。

### QSPI 去初始化

```c title="hal_qspi.h"
int hal_qspi_deinit(uint8_t port);
```

*   **功能**：关闭指定 QSPI 端口，清零时钟分频、控制、数据寄存器，释放硬件资源，用于系统低功耗、模块重启场景。
*   **参数说明**：
	* `port`：目标 QSPI 控制器端口编号。

*   **返回值**：0 表示操作成功，非 0 表示操作失败。
*   **调用说明**：执行后端口时钟关闭，再次使用需重新初始化。

### 单字节（8bit）数据发送

```c title="hal_qspi.h"
int hal_qspi_write_8(uint8_t port, uint8_t data);
```

*   **功能**：通过指定 QSPI 端口发送单字节数据，内部轮询等待硬件忙状态，控制器空闲后写入数据。
*   **参数说明**：
	* `port`：QSPI 控制器端口编号。
	* `data`：待发送的 8 位字节数据。

*   **返回值**：0 表示操作成功，非 0 表示操作失败。
*   **调用说明**：阻塞式发送，常用于发送设备指令、寄存器地址、单字节控制命令。

### 半字（16bit）数据发送

```c title="hal_qspi.h"
int hal_qspi_write_16(uint8_t port, uint16_t data);
```

*   **功能**：发送 16 位半字数据，自动适配硬件 FIFO 位宽，等待硬件空闲后执行写入操作。
*   **参数说明**：
	* `port`：QSPI 控制器端口编号。
	* `data`：待发送的 16 位数据。

*   **返回值**：0 表示操作成功，非 0 表示操作失败。

### 单字（32bit）数据发送

```c title="hal_qspi.h"
int hal_qspi_write_32(uint8_t port, uint32_t data);
```

*   **功能**：发送 32 位单字数据，匹配硬件原生 FIFO 传输位宽，是高速通信的基础接口。
*   **参数说明**：
	* `port`：QSPI 控制器端口编号。
	* `data`：待发送的 32 位字节数据。

*   **返回值**：0 表示操作成功，非 0 表示操作失败。
*   **调用说明**：适用于 Flash、高速外设等 32 位对齐的设备，传输效率高于 8/16 位接口。

### 批量连续数据发送

```c title="hal_qspi.h"
int hal_qspi_write_32x2(uint8_t port, uint32_t data1, uint32_t data2);
int hal_qspi_write_32x8(uint8_t port, uint32_t *data_buf);
int hal_qspi_write_32x16(uint8_t port, uint32_t *data_buf);
int hal_qspi_write_32x32(uint8_t port, uint32_t *data_buf);
```

*   **功能**：分别连续发送 2 个、8 个、16 个、32 个 32 位数据，单次触发硬件传输，保证数据连续性，大幅提升批量传输效率。
*   **参数说明**：
	* `port`：QSPI 控制器端口编号。
	* `data1/data2`：2 字发送的独立数据。
	* `data_buf`：批量数据缓冲区首地址。
*   **返回值**：0 表示操作成功，非 0 表示操作失败。
*   **调用说明**：大数据量传输优先使用批量接口，避免循环单次发送带来的时序间隔。

### 带片选控制的发送接口

```c title="hal_qspi.h"
int hal_qspi_write_8_cs(uint8_t port, uint8_t data, uint32_t cs);
int hal_qspi_write_16_cs(uint8_t port, uint16_t data, uint32_t cs);
int hal_qspi_write_32_cs(uint8_t port, uint32_t data, uint32_t cs);
int hal_qspi_write_32x2_cs(uint8_t port, uint32_t data1, uint32_t data2, uint32_t cs);
int hal_qspi_write_32x8_cs(uint8_t port, uint32_t *data_buf, uint32_t cs);
int hal_qspi_write_32x16_cs(uint8_t port, uint32_t *data_buf, uint32_t cs);
int hal_qspi_write_32x32_cs(uint8_t port, uint32_t *data_buf, uint32_t cs);
```

*   **功能**：在发送数据前自动拉低指定片选信号，发送完成后释放片选，实现多外设总线分时通信。

*   **参数说明**：

	* `cs`：片选信号标识，取值由板卡硬件定义，用于选择总线上的目标外设。

*   **调用说明**：多设备挂载同一 QSPI 总线时，必须使用带`_cs`后缀的接口，防止总线冲突。

	



!!! info "硬件通用限制说明"

    *   QSPI 为同步串行主机模式，不支持从机模式，通信时钟必须在外部设备支持的频率范围内。
    *   所有发送接口均为阻塞式，内部自动轮询硬件忙标志，等待传输完成后返回。
    *   片选信号由硬件驱动自动控制，无需用户手动操作 IO 电平。
    *   驱动为轮询实现，暂不支持中断、DMA 传输模式。
    *   端口、片选的有效取值，由当前运行的硬件平台决定。

	

!!! info "通用使用规则"

    *   **标准通信流程**：`hal_qspi_init`初始化 → 调用对应发送接口传输数据。
    *   **时钟配置规则**：分频值越小，通信时钟越快，需严格匹配外接设备的最高时钟参数。
    *   **总线使用规则**：多外设场景必须使用带片选的接口，避免多设备同时占用总线。
    *   **传输效率规则**：批量数据优先使用`32xN`系列接口，减少硬件触发次数，提升传输速率。
