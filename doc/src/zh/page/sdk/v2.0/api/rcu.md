# RCU API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 RCU 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成系统时钟、复位控制、时钟分频、状态获取等系统核心控制。



## 概述
### 版本特性

SDK 2.0 版本的 RCU 驱动基于统一 HAL_RCU 接口实现，兼容标准 HAL 开发习惯。RCU为芯片**复位与时钟控制单元**，负责全局时钟使能、时钟分频、系统复位管理、工作状态监测。驱动接口精简、执行高效、上电即可使用，是整个系统运行的基础核心模块。

### 适用范围

适用于 SDK 支持的所有板卡，包括：

* 系统时钟源选择与使能控制
* 外设时钟分频配置
* 全局复位/局部复位控制
* 系统工作状态获取
* 低功耗时钟管理
* 外设时钟开关控制



## 头文件与依赖
### 核心头文件

```c title="rcu.c"
#include "hal_rcu.h"
```

*   **hal_rcu.h**：对外暴露标准 HAL_RCU 操作接口，包含控制配置、时钟分频、状态获取等核心函数。

### 编译依赖

*   **CONFIG_DRIVER_RCU**：该宏用于在 Kconfig 中开启 RCU 驱动编译，RCU 为系统核心模块，默认开启。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于时钟分频计算。



## 核心数据类型

RCU 为芯片底层系统单元，接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 32 位数据类型，稳定、通用。



## HAL_RCU 标准接口（hal_rcu.h）
### 设置 RCU 控制寄存器

```c title="hal_rcu.h"
void hal_rcu_set_ctrl(uint32_t val);
```

*   **功能**：配置 RCU 全局控制寄存器，用于使能时钟模块、开启复位功能、选择时钟源、控制外设时钟开关等系统级操作。
*   **参数说明**：
	* `val`：32 位控制配置值，位定义由硬件板卡决定。
*   **调用说明**：
	* 系统启动早期调用，用于配置系统工作模式。
	* 典型使能值：`0b1011`（时钟使能 + 复位控制 + 外设时钟开启）。
	* 配置后立即生效，直接影响系统全局时钟与复位状态。

### 设置 RCU 时钟分频系数

```c title="hal_rcu.h"
void hal_rcu_set_rdiv(uint32_t val);
```

*   **功能**：设置系统时钟分频值，用于将输入时钟分频，生成系统内核、外设所需的稳定工作时钟。
*   **参数说明**：
	* `val`：分频系数。
*   **调用说明**：
	* 分频值决定系统/外设运行频率。
	* 分频设置需与 `hal_rcu_set_ctrl` 配合使用。
	* 示例：`hal_rcu_set_rdiv(256 - 1)` 配置 256 分频。



### 获取 RCU 状态寄存器

```c title="hal_rcu.h"
uint32_t hal_rcu_get_stat(void);
```

*   **功能**：读取 RCU 状态寄存器，获取当前系统时钟状态、复位标志、时钟稳定标志、忙碌状态等系统核心状态。
*   **参数说明**：
	* 32 位无符号整数，每一位代表一种系统状态。
*   **调用说明**：
	* 可用于复位原因判断（上电复位 / 软复位 / 看门狗复位）。
	* 可判断时钟是否稳定、PLL 是否锁定。
	* 常用于系统自检、启动状态检测。




!!! info "硬件通用限制说明"

    *   RCU 是芯片**核心系统单元**，所有外设时钟、复位、时钟源均由其统一管理。
    *   时钟分频、控制配置**立即生效**，错误配置可能导致系统死机或时钟异常。
    *   状态寄存器包含复位标志、时钟锁定、系统状态等关键信息。
    *   无需初始化，上电即处于默认工作状态。




!!! info "通用使用规则"

    *   **调用时机**：RCU 配置应在系统启动最早期执行，优先于所有外设初始化。
    *   **配置流程**：hal_rcu_set_ctrl → hal_rcu_set_rdiv → hal_rcu_get_stat 校验状态。
    *   分频值通常按 “目标分频 - 1” 写入，具体以板卡手册为准。
    *   状态寄存器可用于判断系统异常复位原因，便于调试定位问题。
