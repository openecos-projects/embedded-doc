# RNG API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 RNG 随机数发生器驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成硬件随机数发生器配置、种子设置、随机数获取等功能。

## 概述
### 版本特性

SDK 2.0 版本的 RNG 驱动基于统一 HAL_RNG 接口实现，兼容标准 HAL 开发习惯。驱动支持硬件随机数发生器使能、种子配置、随机数值读取，具备上电即用、接口精简、输出稳定等特点，适用于安全加密、身份识别、随机算法、防护跳频等场景。

### 适用范围

适用于 SDK 支持的所有板卡上的硬件随机数发生场景，包括：

* 数据加密密钥生成
* 设备唯一识别码随机因子
* 算法随机序列生成
* 安全启动、身份认证
* 通信跳频、抗干扰随机时序



## 头文件与依赖
### 核心头文件

```c title="rng.c"
#include "hal_rng.h"
```

*   **hal_rng.h**：对外暴露标准 HAL_RNG 操作接口，包含控制、种子设置、随机数获取等核心函数。

### 编译依赖

*   **CONFIG_DRIVER_RNG**：该宏用于在 Kconfig 中开启 RNG 驱动编译，不定义则不编译。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于 RNG 时钟同步。



## 核心数据类型

RNG 驱动为硬件随机数模块，接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 32 位数据类型，可直接调用。



## HAL_RNG 标准接口（hal_rng.h）

### 设置 RNG 控制寄存器

```c title="hal_rng.h"
void hal_rng_set_ctrl(uint32_t val);
```

*   **功能**：配置 RNG 硬件控制器，用于模块使能、工作模式选择、生成电路控制等。
*   **参数说明**：
	* `val`：32 位控制字。
*   **调用说明**：
	* 使用 RNG 前必须先调用该接口使能模块。
	* 典型使能值：`1`（开启随机数生成）。

### 设置 RNG 随机种子

```c title="hal_rng.h"
void hal_rng_set_seed(uint32_t val);
```

*   **功能**：设置随机数发生器的种子，用于初始化随机序列起点。
*   **参数说明**：
	* `val`：32 位随机种子，可为任意数值。
*   **调用说明**：
	* 相同种子将生成相同随机序列。
	* 如需不可预测随机数，可使用芯片 ID 或浮点噪声作为种子。

### 获取随机数值

```c title="hal_rng.h"
uint32_t hal_rng_get_val(void);
```

*   **功能**：从硬件 RNG 模块获取当前生成的 32 位随机数。
*   **返回值**：32 位随机数。
*   **调用说明**：
	* 可循环调用获取连续随机序列。
	* 使能并设置种子后即可读取。




!!! info "硬件通用限制说明"

   *   RNG 为硬件伪随机数发生器，种子决定输出序列。
   *   使能后自动运行，无需 CPU 干预。
   *   每次读取都会自动推进随机序列。
   *   不依赖外部信号，纯数字逻辑生成。




!!! info "通用使用规则"

   *   **使用流程**：hal_rng_set_ctrl 使能 → hal_rng_set_seed 种子 → hal_rng_get_val 读取。
   *   种子一旦设定，序列固定；重设种子可重启序列。
   *   用于加密场景建议使用芯片唯一 ID 作为种子。
   *   读取无延时，硬件实时生成。
