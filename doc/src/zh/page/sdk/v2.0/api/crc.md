# CRC API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 CRC 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可配置硬件 CRC 校验模块，完成数据校验、冗余计算、数据完整性检测等功能。



## 概述
### 版本特性

SDK 2.0 版本的 CRC 驱动基于统一 HAL_CRC 接口实现，兼容标准 HAL 开发习惯。驱动支持硬件 CRC 控制器配置、初始值设置、结果异或值设置、数据输入、校验结果读取等完整功能，具备计算速度快、不占用 CPU、稳定可靠等特点，适用于数据传输校验、固件完整性检测、存储数据校验等场景。

### 适用范围

适用于 SDK 支持的所有板卡上的硬件 CRC 校验场景，包括通信数据校验、Flash 数据完整性检查、固件包合法性校验、传感器数据容错检测等。



## 头文件与依赖
### 核心头文件

```c title="crc.c"
#include "hal_crc.h"
```

*   **hal_crc.h**：对外暴露标准 HAL_CRC 操作接口，包含控制寄存器配置、初始值、异或值、数据输入、结果读取等核心函数。



### 编译依赖

*   **CONFIG_DRIVER_CRC**：该宏用于在 Kconfig 中开启 CRC 驱动编译，不定义此宏时，驱动不会被编译进工程。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，部分硬件用于 CRC 时钟同步，保证计算稳定。



## 核心数据类型

CRC 驱动为硬件计算模块，接口高度精简，**无自定义枚举与结构体**，全部使用标准 C 32 位数据类型，简洁高效、调用门槛低。



## HAL_CRC 标准接口（hal_crc.h）
### 设置 CRC 控制寄存器

```c title="hal_crc.h"
void hal_crc_set_ctrl(uint32_t val);
```

*   **功能**：配置 CRC 硬件控制器的工作模式，包括 CRC 位数、输入顺序、输出反转、计算模式、使能控制等。
*   **参数说明**：
	* `val`：32 位控制字，具体位定义由硬件平台决定。

*   **调用说明**：
	* 必须在 CRC 计算开始前配置。
	* 常见配置：使能模块、选择多项式、设置数据格式。
	* 示例：`hal_crc_set_ctrl(0b1001001)` 启动 CRC 计算。

### 设置 CRC 初始值

```c title="hal_crc.h"
void hal_crc_set_init(uint32_t val);
```

*   **功能**：设置 CRC 计算的初始值，决定计算起始状态。
*   **参数说明**：
	* `val`：32 位初始值，常见如 `0x0000`、`0xFFFF`。

*   **调用说明**：
	* 每次新计算任务开始前设置。
	* 不同标准（CRC16/CRC32）使用不同初始值。

### 设置 CRC 结果异或值

```c title="hal_crc.h"
void hal_crc_set_xorv(uint32_t val);
```

*   **功能**：设置 CRC 最终结果的异或值，用于输出结果格式转换。
*   **参数说明**：
	* `val`：32 位异或值，通常为 `0x0000` 或 `0xFFFF`。

*   **调用说明**：
	* 用于匹配不同 CRC 标准的输出格式。
	* 不需要异或时设置为 0。

### 写入 CRC 输入数据

```c title="hal_crc.h"
void hal_crc_set_data(uint32_t val);
```

*   **功能**：向 CRC 硬件模块写入待计算的数据，启动一次计算。
*   **参数说明**：
	* `val`：待计算的 32 位数据。

*   **调用说明**：
	* 可连续多次调用，实现数据流累加计算。
	* 硬件自动完成移位、异或多项式计算。

### 获取 CRC 计算结果

```c title="hal_crc.h"
uint32_t hal_crc_get_val(void);
```

*   **功能**：读取当前 CRC 硬件计算完成后的最终校验值。

*   **返回值**：32 位无符号整数，为当前数据的 CRC 结果。
*   **调用说明**：
	* 所有数据写入完成后调用。
	* 读取后可用于校验、存储、打印。





!!! info "硬件通用限制说明"

    *   CRC 为**硬件加速计算**，不占用 CPU 运算资源。
    *   支持 8/16/32 位多种 CRC 标准，由 `ctrl` 寄存器决定。
    *   可连续输入数据流，自动进行链式计算。
    *   计算过程无需干预，结果实时可读。

	

	

!!! info "通用使用规则"

    *   **标准流程**：设置 ctrl → 设置 init → 设置 xorv → 写入 data → 读取 val。
    *   每次新计算任务需重新设置初始值。
    *   连续计算时只需不断写入数据即可自动累加。
    *   不同 CRC 标准需匹配不同 init、xorv、ctrl 配置。
