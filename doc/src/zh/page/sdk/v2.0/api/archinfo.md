# ARCHINFO API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 ARCHINFO 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨板卡兼容性。ARCHINFO 模块用于访问芯片内部系统配置寄存器与芯片 ID 寄存器，提供系统配置读写、芯片唯一标识读写、系统信息打印等基础系统功能，是整个芯片的基础信息管理单元。



## 概述
### 版本特性

SDK 2.0 版本的 ARCHINFO 驱动基于统一 HAL 接口实现，接口简洁、无依赖、轻量化，可在系统启动早期直接调用。驱动支持系统配置寄存器（SYS）读写、芯片 ID 低 32 位（IDL）读写、芯片 ID 高 32 位（IDH）读写，以及完整系统信息打印，可用于芯片识别、系统配置、版本管理、设备唯一标识等关键系统服务。

### 适用范围

适用于 SDK 支持的所有板卡，包括：

* 芯片系统配置寄存器读写与状态查询
* 芯片唯一 ID（IDL + IDH）获取与设置
* 系统启动信息打印、调试信息输出
* 设备身份识别、序列号管理
* 芯片版本、平台信息区分



## 头文件与依赖
### 核心头文件

```c title="archinfo.c"
#include "hal_archinfo.h"
```

*   **hal_archinfo.h**：对外暴露标准 HAL_ARCHINFO 操作接口，包含信息打印、系统寄存器读写、芯片 ID 读写等全套函数。
*   依赖模块：sys_uart（用于打印输出）、log 系统（日志输出）。



### 编译依赖

*   **CONFIG_DRIVER_ARCHINFO**：在 Kconfig 中开启 ARCHINFO 驱动编译，默认开启。
*   **CONFIG_DRIVER_SYS_UART**：系统调试串口，用于信息打印输出。
*   **CONFIG_LOG**：日志系统开关，用于输出调试信息。



## 核心数据类型

ARCHINFO 模块为系统基础寄存器操作接口，**无自定义枚举与结构体**，全部使用标准 C 语言内置类型，可在系统任何阶段调用。



## HAL_ARCHINFO 标准接口（hal_archinfo.h）
### 打印系统架构信息

```c title="hal_archinfo.h"
void hal_archinfo_info(void);
```

*   **功能**：通过系统调试串口打印完整的芯片架构信息，包括当前 SYS 寄存器值、IDL 寄存器值、IDH 寄存器值，用于系统调试、启动信息展示、硬件状态确认。
*   **参数说明**：无参数。

*   **调用说明**：
	* 可在系统初始化完成后直接调用。
	* 常用于启动流程、测试用例、问题定位场景。
	* 与 log 系统配合输出格式化信息。

### 设置系统配置寄存器

```c title="hal_archinfo.h"
void hal_archinfo_set_sys(uint32_t val);
```

*   **功能**：向芯片内部 **SYS 系统配置寄存器** 写入 32 位配置值，用于配置芯片全局工作模式、功能开关、系统状态等。
*   **参数说明**：
	* `val`：要写入 SYS 寄存器的 32 位配置值，每一位对应硬件系统配置。

*   **调用说明**：
	* 写入后硬件立即生效，会影响芯片整体的工作状态。
	* 需根据硬件手册定义正确填写，错误配置可能导致系统异常。

### 设置芯片 ID 低 32 位

```c title="hal_archinfo.h"
void hal_archinfo_set_idl(uint32_t val);
```

*   **功能**：向芯片 **IDL寄存器** 写入 32 位数值，用于设置芯片唯一标识的低 32 位。
*   **参数说明**：
	* `val`：写入 IDL 寄存器的 32 位数值。

*   **调用说明**：
	* 可用于设置设备序列号、硬件标识。
	* 部分芯片该寄存器为只读，写入不生效，具体以使用的硬件为准。

### 设置芯片 ID 高 32 位

```c title="hal_archinfo.h"
void hal_archinfo_set_idh(uint32_t val);
```

*   **功能**：向芯片 **IDH寄存器** 写入 32 位数值，用于设置芯片唯一标识的高 32 位。
*   **参数说明**：
	* `val`：写入 IDH 寄存器的 32 位数值。

*   **调用说明**：
	* 与 IDL 组合构成 **64 位唯一芯片标识符**。
	* 可用于设备认证、加密、身份识别。

### 获取系统配置寄存器值

```c title="hal_archinfo.h"
uint32_t hal_archinfo_get_sys(void);
```

*   **功能**：读取当前 **SYS 系统配置寄存器** 的实时值，用于查询系统当前工作模式、状态。

*   **返回值**：32 位无符号整数，表示当前系统配置状态。
*   **调用说明**：
	* 可用于配置校验、状态自检、调试查询。
	* 读取实时硬件值，不受软件缓存影响。

### 获取芯片 ID 低 32 位

```c title="hal_archinfo.h"
uint32_t hal_archinfo_get_idl(void);
```

*   **功能**：读取芯片 **IDL 寄存器** 数值，获取芯片唯一标识低 32 位。

*   **返回值**：32 位芯片 ID 低位。
*   **调用说明**：
	* 上电固定，通常为芯片出厂唯一标识。
	* 可用于设备区分、序列号生成、授权管理。

### 获取芯片 ID 高 32 位

```c title="hal_archinfo.h"
uint32_t hal_archinfo_get_idh(void);
```

*   **功能**：读取芯片 **IDH 寄存器** 数值，获取芯片唯一标识高 32 位。

*   **返回值**：32 位芯片 ID 高位。
*   **调用说明**：
	* 与 IDL 寄存器组合为 64 位唯一硬件标识符。
	* 用于云端绑定、设备认证、安全启动。





!!! info "硬件通用限制说明"

    *   ARCHINFO 操作芯片**系统级寄存器**，不属于普通外设，不占用 IO、中断、DMA 资源。
    *   SYS、IDL、IDH 均为 **32 位只读 / 可写寄存器**，具体权限由硬件定义。
    *   无需初始化、无需使能，**上电即可直接调用**。
    *   读写操作均为直接访问硬件寄存器，无软件缓存。

	

	

!!! info "通用使用规则"

    *   **调用时机**：可在系统启动任意阶段使用，推荐在 board_init 后立即调用。
    *   **ID 使用规则**：IDL + IDH 组合为 64 位唯一芯片标识符，用于全局唯一识别。
    *   **SYS 配置规则**：写入前必须确认位定义，错误配置可能导致系统故障。
    *   **调试规则**：调试时优先使用 `hal_archinfo_info()` 打印完整信息，便于快速定位硬件状态。
