# TIMER API 2.0

!!! info "文档说明"

    本文档介绍 SDK 2.0 版本中 TIMER 驱动的统一 HAL 接口。该接口采用硬件抽象层设计，标准化 API 定义，具备跨平台兼容性。开发者无需关注底层寄存器与位操作，直接调用统一接口即可完成高精度延时、系统时基、计时统计等功能。


## 概述
### 版本特性

SDK 2.0 版本的 TIMER 驱动基于统一 HAL_TIMER 接口实现，兼容标准 HAL 开发习惯。驱动支持多硬件定时器切换、微秒/ 毫秒/秒级阻塞延时、32 位系统时基计时、高精度时钟周期统计，所有接口统一规范、参数清晰、可在不同硬件板卡间无缝移植，为系统时序控制、延时操作、性能统计、外设驱动提供稳定底层支撑。

### 适用范围

适用于 SDK 支持的所有硬件上的定时器应用场景，包括微秒级精准延时、毫秒级通用延时、系统运行时间统计、外设时序控制、任务耗时测量、周期触发控制等。



## 头文件与依赖
### 核心头文件

```c title="timer.c"
#include "hal_timer.h"
```

*   **hal_timer.h**：对外暴露标准 HAL_TIMER 操作接口，包含延时函数、系统时基初始化、计时获取等核心函数。

### 编译依赖

*   **CONFIG_DRIVER_TIMER**：该宏用于在 Kconfig 中开启 TIMER 驱动编译，不定义此宏，驱动就不会被编译进工程。
*   **CONFIG_CPU_FREQ_MHZ**：CPU 主频配置，用于定时器时钟分频与延时精度计算，必须正确配置以保证延时与计时准确。



## 核心数据类型

TIMER 驱动接口高度精简，使用标准 C 数据类型，无额外自定义枚举与结构体，保证最小系统启动与高精度计时的可靠性。



## HAL_TIMER 标准接口（hal_timer.h）
### 微秒级精准延时

```c title="hal_timer.h"
uint8_t hal_delay_us(uint8_t timer_id, uint32_t val);
```

*   **功能**：使用指定硬件定时器实现微秒级阻塞式精准延时，延时精度由硬件定时器保证，不受编译器优化影响。
*   **参数说明**：
	* `timer_id`：硬件定时器编号，用于选择使用哪一路定时器，有效值由具体硬件决定。
	* `val`：延时微秒数，范围由具体硬件决定。

*   **返回值**：0 表示操作成功，1 表示定时器编号非法、操作失败。
*   **调用说明**：
	* 阻塞式延时，延时期间 CPU 等待，适合时序要求严格的场景。
	* 底层自动根据 CPU 主频将时间转换为硬件计数，保证不同平台延时长度一致。
	* 是毫秒、秒级延时的底层基础接口。

### 毫秒级通用延时

```c title="hal_timer.h"
uint8_t hal_delay_ms(uint8_t timer_id, uint32_t val);
```

*   **功能**：基于微秒延时封装，实现毫秒级阻塞延时，适用于人机交互、外设上电等待、常规时序控制等场景。
*   **参数说明**：
	* `timer_id`：硬件定时器编号。
	* `val`：延时毫秒数。

*   **返回值**：0 成功，1 失败。
*   **调用说明**：
	* 内部直接调用 `hal_delay_us(timer_id, val * 1000)` 实现。
	* 延时精度与微秒级一致，适合中等长度等待。

### 秒级长延时

```c title="hal_timer.h"
uint8_t hal_delay_s(uint8_t timer_id, uint32_t val);
```

*   **功能**：基于毫秒延时封装，实现秒级阻塞延时，适用于系统启动等待、长时间间隔控制等场景。
*   **参数说明**：
	* `timer_id`：硬件定时器编号。
	* `val`：延时秒数。

*   **返回值**：0 成功，1 失败。
*   **调用说明**：
	* 内部调用 `hal_delay_ms(timer_id, val * 1000)` 实现。
	* 适合系统级长时延时控制。

### 系统时基初始化

```c title="hal_timer.h"
uint8_t hal_sys_tick_init(uint8_t timer_id);
```

*   **功能**：初始化指定硬件定时器为 32 位系统时基，配置为连续向上计数模式，作为系统全局运行时间基准。
*   **参数说明**：
	* `timer_id`：用作系统时基的硬件定时器编号。
*   **返回值**：0 成功，1 失败。
*   **调用说明**：
	* 系统启动早期调用一次，用于持续记录系统运行时间。
	* 定时器配置为最长周期（0xFFFFFFFF），不产生中断，后台持续运行。
	* 为 `hal_get_sys_tick` 提供计时基础。

### 获取系统运行时基计数

```c title="hal_timer.h"
uint32_t hal_get_sys_tick(uint8_t timer_id);
```

*   **功能**：读取指定系统时基定时器的当前计数值，返回从时基初始化以来累计的时钟周期数。
*   **参数说明**：
	* `timer_id`：已初始化的系统时基定时器编号。
*   **返回值**：32 位无符号时钟周期计数值。
*   **调用说明**：
	* 可用于任务耗时统计、性能测量、时间戳记录、超时判断等。
	* 计数值单位为 CPU 时钟周期，精度极高。
	* 底层自动将硬件递减计数转换为软件递增计数，便于使用。



!!! info "硬件通用限制说明"

    *   定时器延时为**阻塞式**，延时期间 CPU 不执行其他任务。
    *   所有延时精度依赖 `CONFIG_CPU_FREQ_MHZ` 配置正确，否则时间会偏移。
    *   系统时基定时器一旦初始化，建议全程运行，不与延时功能复用，避免计时错乱。
    *   可用 `timer_id` 数量由平台硬件决定，通常支持 0、1 两路硬件定时器。
    *   不支持中断模式、脉冲输出、输入捕获等高级功能，仅提供基础延时与计时。

	

!!! info "通用使用规则"

    *   **延时流程**：直接调用 `hal_delay_us/hal_delay_ms/hal_delay_s`，无需提前初始化定时器。
    *   **系统时基流程**：`hal_sys_tick_init` 初始化 → 任意时刻调用 `hal_get_sys_tick` 读取计时。
    *   **定时器资源**：延时与时基尽量使用不同 `timer_id`，避免资源冲突。
    *   **精度保证**：微秒级延时适合高频时序控制，毫秒/秒级适合常规等待。
